## CIS 5450 大数据分析 - 期末项目

### 项目目标
展示能够将课程所学应用到真实场景中。可以使用超出课程范围的概念，但项目主体必须基于课程内容。不允许复用其他课程的项目。

### 团队要求
- 4人一组（仅允许一组5人，先到先得）
- 组内不允许与外部合作
- 需考虑时区兼容性
- 通过 Ed Discussion 找队友

### 项目必须包含的组成部分
1. 介绍/背景
2. EDA（探索性数据分析）
3. 建模
4. 挑战/障碍描述
5. 未来方向

### Proposal 要求（5分）- 3月26日截止
通过 Gradescope 提交，需包含：
1. **组员姓名与分工** — 每人负责什么
2. **数据来源** — 需满足数据集大小要求
3. **目标与价值主张** — 研究什么、为什么有趣
4. **建模计划** — 分类 vs 回归？目标变量是什么？
5. **假设检验** — 零假设、检验统计量、是否用模拟
6. **预期挑战**

注意：只需一人提交，但必须在 Gradescope 添加队友。每次重新提交都要重新添加队友，否则全队扣5分。

### 最终提交（123分）- 4月28日截止

**Deliverable 1: 带注释的 Notebook（93分）**
- 一份完整 notebook，包含所有代码
- 用 Markdown 分节，类似作业格式

**Deliverable 2: 演讲（20分）**
- 录制 8-10 分钟演讲（不多不少）
- 内容：目标、数据集、EDA亮点（3-5张图）、建模结果、假设检验结果、洞察、挑战/未来工作
- 要求：用幻灯片、提交 PDF、不要放代码、必须露脸、不能用 TTS
- 可选择现场演讲

**Deliverable 3: 难度概念定位 Notebook**
- 说明每个难度概念在哪里使用及原因

### 数据集要求
- **行数**: 清洗后 50,000+ 行
- **特征**: 至少 7-10 列（灵活）
- **数据必须免费、公开、合法**（不能用公司内部数据或 Kaggle 竞赛专用数据）
- 以结构化数据为主，避免大量 NLP 和 CV

### 成功技巧
- 尽早开始
- 下载数据集（防止 API 失效）
- 边做边注释 notebook
- Google Colab 不支持同时编辑，尽早确定数据清洗流程
- 大数据集可导出清洗后的中间数据
- 明确回答什么问题、怎么回答、有什么收获
