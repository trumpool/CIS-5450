The purpose of the presentation is to give you a chance to practice communicating your data science project to stakeholders, which is an invaluable skill to have. We can spend hours cleaning the data and fine-tuning the models, but, if potential stakeholders cannot understand the value of the results or use the results to formulate actionable strategies, the modeling effort may not receive the attention it deserves.

This is an incredibly challenging skill to master. For this project, it may be helpful to think of a potential stakeholder that may find your project interesting or impactful. And, you have 10 mins to pitch the problem statement, method/model, and implications/finding/results to them!

X. Dataset Size Requirements

Rows:
A large dataset should have 50,000+ rows. Please note that this is the threshold for the number of rows which should be a part of the analysis, so rows removed during data cleaning should not be counted. You should have 50,000+ rows after the data cleaning.

Features:
Additionally, your dataset should have enough interesting features to look at apart from the target classification/regression feature. We can not give a number for this as sometimes a few features can hold a rich amount of information and sometimes we need more features to get a complete picture. Generally, you should have at least 7-10 columns. But this number is more flexible and can vary widely.



XI. Tips for Success
Please get started early, and feel free to come to office hours or post on Ed with any questions.
Once you’ve locked down a dataset, DOWNLOAD it. We had students scraping data from the web in the previous semesters and lost access to the data because the API was updated and no longer compatible with their package.
Comment and annotate your notebook as you go. This will save you a lot of time in the end. 
Despite the name, Google Colab does not allow simultaneous edits on the same notebook. The best way to approach this is:
Try to finalize data cleaning/data wrangling/feature engineering as early as possible, so you all can compile a standardized notebook that is used for modeling.
You can copy blocks of codes in Google Colab. (Ask a TA if you don’t know how to do this.)
If your dataset is large, you are encouraged to download your “cleaned data” and import it midpoint to avoid having to re-process the data every time you restart the session.
Note: You still need to submit all the codes you used for data gathering (if applicable), data cleaning, data integration, data wrangling, and EDA.
As you work on the project, be clear on what question you are answering, what you are doing to answer it, and what your takeaways are.
Email Etiquette Reminder: When communicating with your TA about the project, make sure you cc all your teammates to ensure everyone is on the same page.
